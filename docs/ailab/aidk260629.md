## 操作规范
<br>
敬请按照以下要求操作开发板：

- 🚫 **禁止：水杯、饮料瓶等放在桌上**。以免液体泼洒导致开发板损坏。
    
    可放在实验室四周或地上或书包中。

- 🚫 **禁止：电源线、网线等，从桌子四周穿到桌面上**。以免磕碰导致开发板跌落损坏。

    从桌子中间空洞穿到桌面上。

- 🚫 **禁止：开机状态直接拔电源断电**。以免开发板意外损坏。

    可先按关机键关机。确认关机后再拔电源断电。

- ✴️ **书包等物品远离开发板**。以免磕碰导致开发板跌落损坏。<br>

    可放在实验室四周或地上。

**‼️未按操作规范使用开发板导致损坏的，要照价赔偿‼️**

<br>

或在PC（个人电脑）的终端 PowerShell 中执行：

```bash
ssh HwHiAiUser@192.168.137.100
```

在屏幕提示信息 `HwHiAiUser@192.168.137.100's password: ` 后面输入密码 `Mind@123`，输完后按 `回车` 键。**✳️ 输入密码过程中，在屏幕上不会显示信息，这是正常的（因为是密码，所以不能显示出来被TA人看到）**

如果遇到以下报错信息：

```bash
~ % ssh HwHiAiUser@192.168.137.100
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@    WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!     @

...
```

可以先执行以下命令：

```bash
ssh-keygen -R 192.168.137.100
```

然后再尝试 `ssh HwHiAiUser@192.168.137.100`

## 操作建议
<br>
有以下操作建议，供同学们参考。

### 创建目录相关
<br>
按实验手册创建目录：

```bash
mkdir ~/compiler
```

如果遇到类似报错信息如下：

```bash
mkdir: cannot create directory ‘/home/HwHiAiUser/complier’: File exists
```

可以先删除目录：

```bash
rm -r ~/compiler
```

然后尝试重新创建目录。

### 上传文件到开发板
<br>
可以有几种方式。比如：

- 用 MobaXterm 软件传文件。详见：[MobaXterm指南-传文件↗]

- 或者在PC（个人电脑）的终端 PowerShell 中敲命令传文件。详见：[Linux指南-scp远程复制文件/目录↗]。比如：

    先进入下载的压缩包的保存目录：（假定保存到 PC 用户的 HOME 目录的 tmp26/06 子目录中）

    ```bash
cd ~/tmp26/06
    ```

    然后用 scp 传文件到开发板上：

    ```bash
scp BiShengCompiler-5.1.0.2-aarch64-linux.tar.gz HwHiAiUser@192.168.137.100:/home/HwHiAiUser/compiler
    ```

### 编辑文件
<br>
实验手册要求修改汇编代码，可以有几种方式。比如：

- 在开发板上用 vim 修改，详见：[Linux指南-vim文本编辑↗]
- 或者，用 PC（个人电脑）上的 VSCode 直接编辑开发板上的文件，详见：[VSCode指南-远程连接↗]

### 设置环境变量
<br>
可复制以下命令替代实验手册中的设置命令。**✳️ 新开一个终端登录开发板时，都要设置一次环境变量。**

```bash
export PATH=~/compiler/BiShengCompiler-5.1.0.2-aarch64-linux/bin:$PATH
export LD_LIBRARY_PATH=~/compiler/BiShengCompiler-5.1.0.2-aarch64-linux/lib:~/compiler/BiShengCompiler-5.1.0.2-aarch64-linux/lib/aarch64-unknown-linux-gnu:$LD_LIBRARY_PATH
```

### 多人同时使用开发板（昇腾）
<br>
可通过以下方式，实现多人同时使用开发板。

1. **插实验室网线**

    插实验室网线，到开发板上下排列的2个网口的下面👇那个网口。可获得一个校园网 IP 地址。

2. **查看第二个 IP 地址**

    ```bash
ifconfig | grep 172
    ```

    如下类似信息中的 `172.18.145.228`，就是开发板的校园网 IP 地址。

    ```bash
    inet 172.18.145.228  netmask 255.255.255.0  broadcast 172.18.145.255    
    ```    

3. **PC（个人电脑）连接开发板**

    PC（个人电脑）先连校园网，比如连接 JNU-WLAN。

    假设开发板的第二个 IP 地址是： 172.18.145.228。
    
    然后在 PC（个人电脑）的 终端（Powershell）中，ping 开发板的 172 开头的 IP 地址。比如：

    ```bash
ping 172.18.145.228
    ```

    如果无法 ping 通，请联系实验室老师协助。

    接下来在PC（个人电脑）的 终端（Powershell）中，ssh 登录开发板。比如：

    ```bash
    ssh HwHiAiUser@172.18.145.228
    ```

4. **新建实验目录**

    新建和 compiler 不一样的目录，比如 compiler1（或 compiler2，或 compiler3，等）。假设新建 compiler1：

    ```bash
mkdir ~/compiler1
    ```

5. **按实验手册做实验**

    上传的目录，以及环境变量设置，将 compiler 改成 compiler1（或 compiler2，或 compiler3，等）即可。


### 多人同时使用开发板（鲲鹏）
<br>
可通过以下方式，实现多人同时使用开发板。

1. **连接实验室 WiFi b102**

    连接实验室 WiFi b102。详见：[鲲鹏开发板指南-连WiFi↗]

2. **查看第二个 IP 地址**

    ```bash
ifconfig | grep 192
    ```

    如下类似信息中的 `192.168.31.80`，就是开发板的第二个 IP 地址。

    ```bash
    inet 192.168.137.100  netmask 255.255.255.0  broadcast 192.168.137.255
    inet 192.168.31.80  netmask 255.255.255.0  broadcast 192.168.31.255  
    ```    

3. **PC（个人电脑）连接开发板**

    PC（个人电脑）先连实验室 WiFi b102。

    假设开发板的第二个 IP 地址是： 192.168.31.80。
    
    然后在 PC（个人电脑）的 终端（Powershell）中，ping 开发板的第二个 IP 地址。比如：

    ```bash
ping 192.168.31.80
    ```

    如果无法 ping 通，请联系实验室老师协助。

    接下来在PC（个人电脑）的 终端（Powershell）中，ssh 登录开发板。比如：

    ```bash
    ssh HwHiAiUser@192.168.31.80
    ```

4. **新建实验目录**

    新建和 compiler 不一样的目录，比如 compiler1（或 compiler2，或 compiler3，等）。假设新建 compiler1：

    ```bash
mkdir ~/compiler1
    ```

5. **按实验手册做实验**
